4. Developer Technical Specifications
Tech Stack:

Frontend: Next.js (React) – for server-side rendering/static generation (SSG) that boosts SEO and performance
. Next.js provides automatic code-splitting and image optimization. Use React components for each section.

Styling: Tailwind CSS (utility-first), or styled-components. Tailwind yields fast dev with consistent spacing. Alternatively, a CSS framework like Bootstrap (dark theme) could work, but Tailwind is more modern.

State/Interactions: Minimal – likely only for form submission or menu toggle. No heavy frameworks needed beyond React.

Backend/API: None needed for static site. For contact form, either a third-party form service (Formspree, EmailJS) or a simple serverless function (Node/Express, or Next.js API route) to send email. If using Netlify, Netlify Forms is an option.

Hosting/CD: Deploy on Vercel (native for Next.js, supports SSR/ISR, custom domains) or Netlify. Both offer CI/CD from Git. Setup GitHub Action or Vercel auto-deploy on main branch commits.

Performance Optimizations:

Use Next.js <Image> component for responsive images (serves WebP/AVIF, lazy-loads by default).
Preload key resources (fonts, critical CSS). Minify CSS/JS (Next does this by default in production).
Purge unused CSS (Tailwind purge).
Defer non-critical JS.
CDN: Vercel’s global CDN; ensure stale-while-revalidate caching for images.
Reduce number/size of dependencies. Avoid large UI libs; only include necessary icons/fonts.
Core Web Vitals: Aim for LCP < 2.5s (fast hero), CLS minimal (avoid layout shifts by sizing images and using stable elements), FID small (minimize JS).
